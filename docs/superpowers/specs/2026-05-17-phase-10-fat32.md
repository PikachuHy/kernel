# Phase 10: FAT32 Filesystem — Design Spec

## Overview

FAT32 filesystem server as a ring-3 process. Reads disk sectors via `/dev/ahci0`
Channel (devfs wraps BlockDevice). Parses BPB, FAT table, root directory, and
cluster chains to resolve file paths and read file data.

**Architecture**: IPC-native — FAT32 server opens `/dev/ahci0` like any other file,
reads raw sectors, and presents files as Channels to clients.

## Components

### 1. devfs block device handler (`handle_block`)

New device handler in devfs that wraps a `BlockDevice` as a Channel.

- `open("/dev/ahci0")` → creates Channel, stores BlockDevice pointer in handler context
- `Read(FileMsg{lba, count})` → `blkdev_read(dev, lba, buf, count)` → return data
- `Write(FileMsg{lba, count})` → `blkdev_write(dev, lba, buf, count)` → return status
- `Stat` → returns total sectors and sector size

The block device name is passed as the path after `/dev/` (e.g., `/dev/ahci0` → lookup `blkdev_find("ahci0")`).

### 2. FAT32 server (ring-3 ELF)

**BPB parsing** (sector 0):
- Bytes per sector (offset 11), sectors per cluster (offset 13)
- Reserved sectors (offset 14), number of FATs (offset 16)
- Sectors per FAT (offset 36 for FAT32), root cluster (offset 44)

**FAT table**:
- Located at `reserved_sectors * bytes_per_sector`
- Each entry is 4 bytes (FAT32), values: 0=free, 0x0FFFFFF8+=EOF, else=next cluster

**Cluster chain traversal**:
- Cluster → LBA: `data_start_lba + (cluster - 2) * sectors_per_cluster`
- Follow FAT entries to build chain

**Directory entries** (32 bytes each):
- Name (8.3 format), attributes (0x10 = directory), first cluster (high+low), size
- Long file names (LFN) via special entries — parse for full name

**open path resolution**:
- Start from root cluster (BPB root_cluster for FAT32)
- Walk directory entries, match name, follow cluster chain
- Return file Channel with cluster chain info

**read**:
- Walk cluster chain to find the cluster containing the requested offset
- Read sector(s) from that cluster, return data

### 3. Boot integration

```cpp
// After AHCI init, expose blkdev via devfs:
BlockDevice* dev = blkdev_find("ahci0");
if (dev) {
    // Register a virtual path "/dev/ahci0" that devfs routes to handle_block
    devfs_register_block("ahci0", dev);
}
```

devfs's `_start()` dispatches unknown devices to `handle_block` which looks up
the BlockDevice by name.
