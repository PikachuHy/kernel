# Phase 10: FAT32 — Implementation Plan

> **For agentic workers:** Use subagent-driven-development (recommended) to implement this plan task-by-task.

**Goal:** FAT32 filesystem server as ring-3 process, with devfs block device passthrough.

**Architecture:** devfs wraps BlockDevice as Channel (`/dev/ahci0`). FAT32 server opens it, reads raw sectors, parses BPB/FAT/directory, presents files as Channels.

---

### Task 1: devfs block device handler

**Files:**
- Modify: `kernel/fs/devfs/devfs.cpp`

Add `handle_block` function that wraps a BlockDevice as a file Channel. Modify `_start()` dispatch to route unknown devices to a block device lookup.

`handle_block`:
```cpp
static void handle_block(uint32_t file_chan, const char* dev_name) {
    // Look up BlockDevice by name
    BlockDevice* dev = blkdev_find(dev_name);
    if (!dev) {
        // device not found, send error and close
        FileResponse err = {-1, 0};
        channel_write(file_chan, &err, sizeof(err));
        handle_close(file_chan);
        return;
    }
    // ... read FileMsg, dispatch Read(blkdev_read)/Write(blkdev_write)/Stat/Close
}
```

`_start()` dispatch change: after checking null/zero/console, add a fallback that tries `blkdev_find(rel)`. If found, call `handle_block(file_handle, rel)`.

**Verification**: after AHCI init, `blkdev_register` makes "ahci0" available. Opening `/dev/ahci0` reaches handle_block.

### Task 2: FAT32 server — BPB + FAT parsing

**Files:**
- Create: `kernel/fs/fat32/fat32.cpp`
- Create: `kernel/fs/fat32/fat32.ld`
- Create: `kernel/fs/fat32/BUILD.bazel`

Ring-3 ELF at 0x800000. On startup:
1. `h = sys_open("/dev/ahci0", O_RDONLY)` → get handle
2. Read sector 0 (BPB): send FileMsg::Read(offset=0, length=512), get data
3. Parse BPB fields: bytes_per_sector, sectors_per_cluster, reserved_sectors, sectors_per_fat, root_cluster
4. Validate: bytes_per_sector == 512, FAT32 signature check
5. Log disk info: "FAT32: X MB disk, Y sectors/cluster, root cluster Z"
6. Compute FAT start LBA and data start LBA

**Verification**: FAT32 server starts, prints BPB info.

### Task 3: FAT32 server — directory + file reading

**Files:**
- Modify: `kernel/fs/fat32/fat32.cpp`

Add:
- `read_sectors(handle, lba, count, buf)` — sends Read FileMsg, reads response
- `read_fat_entry(handle, cluster)` — reads FAT table entry for a cluster
- `follow_cluster_chain(handle, start_cluster)` — builds list of clusters
- `read_cluster(handle, cluster, buf)` — reads a cluster's worth of sectors
- `read_directory(handle, cluster)` — reads directory entries from a cluster
- `find_file(handle, root_cluster, name)` — walks directory tree to find file
- `fat32_open(path)` — entry point for open: find file, return handle with cluster chain info

Event loop:
- Read OpenPayload from mount Channel (handle 1)
- Resolve path via `fat32_open`
- Build cluster chain, ack on ack_handle, enter handle_file loop
- handle_file: process Read(offset, length) → use cluster chain to read data

**Verification**: init opens `/kernel.elf` via FAT32, reads first 4 bytes, verifies ELF magic (0x7F 'E' 'L' 'F').
